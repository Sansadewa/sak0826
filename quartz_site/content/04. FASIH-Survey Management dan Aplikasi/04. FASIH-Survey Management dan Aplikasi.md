#### ALUR PEMUTAKHIRAN (1)

Yang harus diperhatikan di wilayah offline tarik sampel:

1. Pemutakhiran hanya dapat dilakukan oleh 1 org petugas (PPL) dan hanya 1 gawai (tidak dapat berganti gawai)
2. Karena penarikan sampel dilakukan oleh PPL maka hasil Tarik sampel (DSRT), otomatis akan langsung terassign di akun PPL tersebut dan hanya di gawai PPL itu. Sehingga pendataan pun juga hanya dapat dilakukan oleh 1 PPL (PML tidak dapat reassign sampel).
3. Hasil pemutakhiran yang sudah dilakukan, tetap harus dan wajib disubmit (dikirim ke server) ketika sudah mendapatkan sinyal atau berada diwilayah yang ada sinyal.
4. Begitu pula dengan pendataan. Rumah tangga sampel yang sudah didata, harus dan wajib disubmit (dikirim ke server) ketika sudah mendapatkan sinyal atau berada di wilayah yang ada sinyal.

#### ALUR MANAJEMEN SURVEI

#### ROLE PADA FASIH (1)

Administrator (Admin)

**Pengelola utama sistem** di level BPS RI/Prov/KabKota.

Admin memiliki tugas untuk **mengalokasikan akun** sesuai dengan peran yang ditetapkan. Selain itu, admin Kab/kota bertugas **mengalokasikan sampel**.

Viewer

Hanya memiliki kewenangan untuk melihat **monitoring dan dashboard progres pendataan**, serta **melihat data hasil pendataan** di FASIH SM.

Viewer tidak dapat melakukan approval terhadap data hasil pencacahan.

PML

Sebagai **pemeriksa** hasil pencacahan yang telah diisi lengkap dan telah di-submit ke server oleh petugas lapangan.

Hasil pemeriksaan dari pemeriksa berupa **approve atau reject data sampel**.

PPL

Petugas lapangan yang bertugas **mewawancarai responden dan menginput data ke Fasih Mobile** serta melakukan **submit data** ke server.

Bertugas memastikan bahwa data sampel yang dikerjakan sudah di*-approve* sampai berstatus complete

#### ROLE PADA FASIH (2)

#### Ketentuan untuk kegiatan SAKERNAS 2026:

- *1. BPS RI** mendaftarkan akun SSO BPS Provinsi sebagai Admin Provinsi pada FASIH SM
- *2. BPS Provinsi** menyerahkan user yang menjadi admin provinsi kepada admin BPS RI. BPS Provinsi mendaftarkan akun SSO BPS Provinsi sebagai viewer provinsi dan akun SSO BPS Kako sebagai admin Kab/Kota.
- *3. BPS Kab/Kota** mendaftarkan akun SSO BPS Kab/Kota sebagai viewer kab/kota. BPS Kab/Kota harus membuat file alokasi sampel petugas dan pemeriksa lapangan sensus.
4. Untuk **user petugas non-organik atau mitra** maka perlu melakukan **pendaftaran terlebih dahulu ke sistem SOBAT**.

### MANAJEMEN PENGGUNA

- *2** BPS RI mengalokasikan admin BPS Provinsi (organik BPS). Admin provinsi hanya dapat dilihat atau diubah oleh admin pusat atau admin provinsi lainnya.
- *3** BPS Provinsi mengalokasikan viewer Provinsi dan admin Kab/Kota (organik BPS). Admin kab/kota hanya dapat dilihat atau diubah oleh admin pusat, admin provinsi, atau admin kabupaten/kota lainnya.

## 4. PEMERIKSAAN DAN PERBAIKAN DATA

#### PEMERIKSAAN DATA DI FASIH-SM

- Pemeriksaan data pada FASIH-SM hanya dapat dilakukan oleh admin menggunakan akun SSO BPS
- Pemeriksaan dilakukan oleh admin BPS Kab/Kota (Fitur akan dibuka sesuai jadwal evaluasi data oleh BPS Kab/Kota)
- Pemeriksaan mencakup kelengkapan isian serta kesesuaian dan kewajaran isian.
- Data yang diperiksa adalah assignment yang sudah di-approve oleh PML
- Data yang sudah diperiksa dapat dilakukan approval (setuju) atau reject (tolak)
- Data yang di-approve oleh admin kab/kota dianggap sebagai data final
- Data yang di-reject selanjutnya akan dikembalikan lagi ke petugas di level bawahnya untuk dilakukan perbaikan

#### FASIH DASHBOARD

### AKSES

- Alamat Dashboard berada pada : [https://fasih-dashboard.bps.go.id](https://fasih-dashboard.bps.go.id/)

Untuk mengakses aplikasi Dashboard, pengguna yang diizinkan untuk login hanya pengguna yang memiliki akun *Single Sign-On* (SSO) BPS.

#### LOGIN

Login dilakukan menggunakan SSO BPS

- SSO Pegawai, untuk pegawai BPS
- SSO eksternal, untuk selain pegawai BPS

Pengguna memilih jenis login SSO, kemudian klik tombol "Go!". Selanjutnya, pengguna akan diarahkan menuju form login SSO BPS.

### NAVIGASI (LANDING PAGE)

Setelah berhasil login, Pengguna akan diarahkan pada landing page aplikasi dashboard.

Pada landing page, terdapat shortcut dashboard yang dapat diakses oleh pengguna, sesuai dengan kewenangannya.

Tersedia pula menu Dashboard, untuk melihat dashboard yang dapat diakses, dalam bentuk list.

#### NAVIGASI (LANDING PAGE)

Pada Menu Dashboard, ditampilkan list dashboard yang dapat diakses oleh pengguna. Pastikan terdapat dashboard Sakernas 2026 Agustus pada list tersebut.

### NAVIGASI (MENU PROFIL)

Selain itu terdapat pula menu settings, yang terdiri atas sub menu :

1. Profil : untuk melihat profil pengguna

2. Info : Untuk melihat informasi akun pengguna

3. Log out

Pada Halaman Profil -> Security Access, Pastikan sudah terdaftar sebagai Roles Sakernas 2026, agar dapat mengakses dashboard Sakernas 2026.

Untuk masuk ke dalam dashboard yang akan dilihat, terdapat dua cara

1. Klik shortcut dashboard pada landing page
2. Klik nama dashboard pada list dashboard di menu "Dashboards"

Kemudian, pengguna akan diarahkan menuju halaman dashboard yang dipilih.

Pada dashboard terdapat tab dan filter.

Tab, Berfungsi untuk menampilkan informasi tertentu, yang berisikan satu atau beberapa chart, berbentuk partisi yang memuat informasi berbeda dengan tab lain dalam satu dashboard.

Dalam satu tab, dapat pula dilakukan partisi informasi menjadi beberapa tab.

Filter, berfungsi untuk melakukan penyaringan informasi yang ditampilkan pada dashboard, sesuai dengan nilai atau range nilai yang ditentukan oleh pengguna.

Filter yang dipilih pengguna, akan berpengaruh pada semua tab dan informasi yang ada pada dashboard tersebut.

Chart Option, berada pada sebelah kanan atas sebuah chart. Chart Option berisi pilihan action yang dapat dilakukan oleh pengguna. Pilihan yang tersedia yaitu,

Force Refresh : Untuk mengambil data paling update dari database (renew cache)

Enter Fullscreen : masuk mode layar penuh

Share : Berbagi link dashboard

Download : mengunduh data yang ditampilkan pada dashboard dalam format image file.

#### KONTEN (PROGRES PENDATAAN)

Fitur monitoring progres pendataan menggambarkan tingkat realisasi dari pendataan responden dari keseluruhan target yang telah ditetapkan. Ditampilkan jumlah target dan realisasi pendataan beserta persentase progres baik secara nasional maupun provinsi. Disediakan juga progres pendataan berdasarkan wilayah provinsi, kabupaten/kota, dan SLS yang menampilkan target sampel, persentase realisasi, serta rumah tangga yang berhasil dicacah

### KONTEN (PROGRES PENDATAAN)

Hasil gambaran kunjungan PPL kepada rumah tangga. Menunjukkan rumah tangga yang telah berhasil untuk dicacah.

### KONTEN (DYNAMIC WEIGHTING)

Fitur dashboard Dynamic Weighting menggambarkan indikator jenis kegiatan, Lapangan Pekerjaan Utama, dan Status pekerjaan utama yang tergaggregasi berdasarkan wilayah provinsi dan kabupaten.

Informasi yang ditampilkan bersumber dari data yang telah di submit ke server dan telah tertimbang, secara dinamis. Sehingga nilainya akan berubah menyesuaikan data yang telah diterima di server. Proses sinkronisasi data dynamic weighting akan dilakukan secara periodik
