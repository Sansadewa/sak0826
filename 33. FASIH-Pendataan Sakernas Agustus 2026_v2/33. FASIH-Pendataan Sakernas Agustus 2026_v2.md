#### Status Assignment:

- *a. Open** : status awal sampel yang dialokasikan kepada PPL dan belum dilakukan submit
- *b. Submit** : status sampel yang sudah dikerjakan oleh PPL yang kemudian di-submit dan mendapat respon sukses dari server
- *c. Reject** : merupakan status sampel yang sudah pernah di-submit ke server tetapi setelah diperiksa ternyata ditolak oleh PML sehingga dikembalikan ke PPL
- *d. Pending** : status sampel yang di-submit oleh PPL tetapi belum mendapat respon dari server karena masih menunggu koneksi internet atau antrian masuk ke server
- *e. Approved** : status sampel yang pernah di-submit oleh PPL ke server dan setelah diperiksa, sudah dinyatakan sesuai oleh PML/admin kab/kota

#### Keterangan:

- A. Tabel Assignment : berisi daftar rumah tangga sampel per blok sensus yang ditugaskan ke PPL/PML
- B. Jumlah Baris : menampilkan jumlah baris sampel pada halaman Daftar Assignment. Jumlah assignment yang ditampilkan per halaman tersedia pada pilihan ini, yaitu 10, 25, 50, dan 100.
- C. Cari (Search) : melakukan pencarian pada tabel assignment dengan kata kunci yang diinginkan
- D. Icon : Menampilkan beberapa fitur yang dapat digunakan oleh petugas seperti filter by status, urutkan, dan info warna assignment

#### Keterangan:

- *Tambah Assignment dan Tandai Wilayah Selesai**: hanya digunakan pada survei tipe pemutakhiran dan tidak digunakan di pendataan
- *Filter By Status**: menampilkan urutan rumah tangga pada BS/NKS berdasarkan status yang dipilih. Pilihan status terdiri atas Open, Pernah dibuka, Submit, Approve, Reject, dan Pending
- *Urutkan**: menampilkan urutan rumah tangga pada BS/NKS berdasarkan kolom yang tersedia yaitu NUS, KRT, FlagPanel, NoBang, Hasil Kunjungan, dan Alamat
- *Info Warna Assignment**: memberikan informasi status assignment berdasarkan warna

- *Daftar Upload** : melihat daftar progress upload assignment
- *Atur Tampilan Kolom** : mengatur kolom-kolom yang ingin ditampilkan pada daftar assignment di antaranya NUS, Nama KRT, FlagPanel, NoBang, Alamat, dan Kunjungan
- *Peta** : menampilkan peta lokasi (tidak digunakan pada pendataan)
- *Tarik Sampel:** Tidak digunakan pada survei pendataan
- *Refresh** : memperbarui status pada tabel assignment

#### Keterangan:

- *Buka** : menampilkan E-form SAKERNAS AGS 2026- PENDATAAN
- *Sinkronisasi Assignment Ini**: khusus untuk assignment **selain** status OPEN/DRAFT fungsinya untuk mendapatkan STATUS dan mengunduh data dari server. Khusus untuk assignment OPEN/DRAFT, fungsinya untuk mendapatkan status assignment tanpa mengunduh data.
- *Assign** : melakukan alokasi sampel ke PPL oleh PML (tidak digunakan pada survei SAKERNAS)
- *Restore**: memulihkan kembali isian data wawancara sesuai kondisi pada versi tanggal wawancara yang dipilih
- *Info Assignment:** menampilkan version code lokal dan server serta ukuran data
- *Hapus** : menghapus assignment sampel yang telah ditambahkan secara manual oleh PPL. Tombol ini hanya berfungsi di survei tipe listing/pemutakhiran

#### Rincian Waktu Wawancara

- *Ambil Waktu**: mengambil waktu wawancara berdasarkan waktu perangkat yang digunakan
- *Perbarui Waktu**: memperbarui waktu wawancara yang sudah diambil/terisi sebelumnya
- *Hapus**: mengosongkan isian waktu wawancara yang sudah diambil sebelumnya

#### Rincian Tagging Lokasi

- *Ambil Lokasi**: mengambil keterangan lokasi petugas berdasarkan titik koordinat latitude, longitude, dan akurasinya
- *Perbarui Lokasi**: memperbarui waktu wawancara yang sudah diambil/terisi sebelumnya
- *Hapus**: mengosongkan isian tagging lokasi wawancara yang sudah diambil sebelumnya
- Tagging lokasi wajib dilakukan untuk seluruh rumah tangga sampel
- Tagging lokasi dapat dilakukan dalam kondisi tidak ada sinyal internet. Silakan pastikan perangkat memiliki GPS, fitur GPS/location sudah diaktifkan, dan berada di ruangan terbuka ketika melakukan tagging.

#### Histori Tagging Lokasi

*Digunakan untuk melihat histori tagging lokasi dan sebagai mitigasi jika terjadi kasus tagging lokasi hilang*

- *Titik kuning**: lokasi koordinat dari hasil tagging yang sudah dilakukan sebelumnya
- *Titik merah**: lokasi petugas saat ini

Untuk kasus tagging lokasi hilang dan ingin mengembalikan isiannya, lakukan:

1. Tap pada titik kuning yang diinginkan sesuai lokasi rumah responden
2. Tap **AMBIL LOKASI** untuk mengembalikan isian tagging lokasi

## SUBMIT / KIRIM DATA KE SERVER

- Apabila seluruh pertanyaan telah terisi dan sudah tidak ada error maka PPL selanjutnya dapat mengirimkan data tersebut ke server dengan melakukan proses **SUBMIT.**
- Proses SUBMIT dilakukan untuk tiap assignment yang sudah selesai didata.
- Tombol SUBMIT akan muncul apabila gawai masing-masing PPL terkoneksi dengan internet

## PERBAIKAN DATA HASIL REJECT

- Data yang sudah berhasil terkirim ke server/SUBMIT selanjutnya akan diperiksa oleh PML
- Jika terdapat ketidaksesuaian data maka PML dapat melakukan REJECT data yang disubmit oleh PPL.
- PPL melakukan Sinkronisasi Data untuk mendapatkan status assignment terbaru sehingga diketahui ruta mana yang di-reject
